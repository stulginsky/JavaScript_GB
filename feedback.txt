.filter метод у массива создает новый массив, внутри функции-обработчика мы должны возвращать true, если значение должно быть в новом массив или false, если значения не должно быть.

const productWithPhoto = products.filter(function(products) {
  return 'photos' in products && products.photos.length !== 0;
});
console.log(productWithPhoto);
Проверяем что существует значение в product.photos и что длина массива больше нуля, в итоге возвращаем true, либо, если что-то не так - false.
Советую использовать стрелочные функции для методов массива, в дальнейшем вы именно стрелочные функции для таких вещей и будете использовать чаще всего.
В сортировке нам надо вернуть отрицательное число, если первое значение (первый параметр) меньше второго, положительное значение надо вернуть, если первый параметр больше второго и 0, если значения равны между собой. Сейчас же возвращается либо -1, либо undefined.
По сортировке - для сравнения чисел можно просто одно из другого вычесть:

products.sort((a, b) => a.price - b.price);
Такое возможно потому что в этой функции, которую передаем в .sort принимается 2 значения из массива (можно считать два случайных значения). При вычитании как раз будет возвращаться либо отриц. число, либо положительное, либо 0, если два элемента равны по цене.
